HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The style element defines CSS styles associated with a webpage.
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • Elements that are nested inside other elements are called child elements.
  • The element that holds the child elements is known as the parent element.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage. (Ex. body element)
  • An example of non-semantic elements are the div element and span element.
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • The h1, h2, h3, (and so on) elements represent the level of heading a given text block represents.
  • The p element represents a paragraph or block of text.
  • The ul, ol, li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about images that are displayed on the webpage.
  • The br element creates a line of empty space, or a line break, bewtween two blocks of content.
  • The a element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link.
  • Section elements are semantic elements that define a section in a document that contains thematic content.

CSS

  • The internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • An external style sheet is more commonly used in order to keep code neat and makes design changes easier.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The border property assigns a border and attributes it a size and colour.
  • The box-shadow CSS property adds shadow effects around an elements frame.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • A declaration contains two important components: the CSS property we want to apply and the value of the property.
  • The period preceding card designates this as a class selector. Any HTML elemetn assigned to the class will gain these property assignments from the CSS.
  • When assigning colours in CSS, one can include semantic references or a hexadecimal code.

Git

  • cd: also known as command, used to access file locations
  • touch: used to create or replace a file easily
  • code .: opens project on default coding program.
  • git status: checks what branch we are currently on
  • git checkout main: changes your working branch to the main branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commits all files in the working branch
  • git commit -m "message here": associates a message with our commit
  • git pull origin main: pulls in the base branch
  • git push origin branchname: pushs newly made changes
  • A pull request or PR is request to review newly changed code in a branch in an attempt to merge with the main code
  • Issues are problems that need to be tackled by the team. It can also be used as updates.

JavaScript

  • A variable or var is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A string is a sequence of one or more characters that may consist of letters, numbers, or symbols.
  • A console.log method is used to output a message to the web console by adding an argument.
  • The word console refers to a test environment that developers use to check out their code.
  • .log() is a set of instructions that can be executed by a computer and will output whatever we add inside the parentheses to the console.
  • An if statement is a way of introducing decision-making into the code. The code will only execute if the statement meets a condition, or is true.
  • The strict equality operator, or === checks to see if two values are equal, and returns a Boolean result 'true' if the values are equal and 'false' if the values are not equal.
  • An array is a single variable that is used to hold a group of data.
  • An index is the location number of the data item in the array.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A for loop contains a starting point, condition, and a statement that allows the array to iterate over each item.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • There are two requirements for a function; define the function and then call the function.
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number.
